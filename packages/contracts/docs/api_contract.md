# API Contract Baseline

All SQ-BI APIs use the `/api/v1` namespace and the same response envelope.

## Response Envelope

```json
{
  "request_id": "req_123",
  "data": {},
  "error": null
}
```

## Error Envelope

```json
{
  "request_id": "req_123",
  "data": null,
  "error": {
    "code": "PERMISSION_DENIED",
    "message": "No permission to access this metric.",
    "details": {}
  }
}
```

## Security Rules

- `POST /api/v1/query/ask` executes only guarded read-only SQL generated from the active Skill/semantic context.
- LLM output may become metric, Skill, report, or guarded query output only.
- Candidate lists must be permission-filtered before display.
- SQL generated by the LLM must pass guardrails before execution.
- Every query result must include `audit_id` and `lineage`.
- `POST /api/v1/exports` accepts immutable query/report snapshots only.
- Export and share APIs must not read source databases directly.
- Share previews expose metadata from captured snapshots only, never live query data.
- Subscription runs execute as the subscriber or an audited service principal.

## Route Ownership

Route ownership is defined in `sq_bi_contracts.api.API_ROUTES`. Worktrees may implement owned routes, but should not change shared envelopes without updating this foundation branch first.
